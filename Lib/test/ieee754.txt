Python IEEE 754 floating point support
======================================

>>> kutoka sys agiza float_info kama FI
>>> kutoka math agiza *
>>> PI = pi
>>> E = e

You must never compare two floats ukijumuisha == because you are sio going to get
what you expect. We treat two floats kama equal ikiwa the difference between them
is small than epsilon.
>>> EPS = 1E-15
>>> eleza equal(x, y):
...     """Almost equal helper kila floats"""
...     rudisha abs(x - y) < EPS


NaNs na INFs
=============

In Python 2.6 na newer NaNs (not a number) na infinity can be constructed
kutoka the strings 'inf' na 'nan'.

>>> INF = float('inf')
>>> NINF = float('-inf')
>>> NAN = float('nan')

>>> INF
inf
>>> NINF
-inf
>>> NAN
nan

The math module's ``isnan`` na ``isinf`` functions can be used to detect INF
and NAN:
>>> isinf(INF), isinf(NINF), isnan(NAN)
(Kweli, Kweli, Kweli)
>>> INF == -NINF
Kweli

Infinity
--------

Ambiguous operations like ``0 * inf`` ama ``inf - inf`` result kwenye NaN.
>>> INF * 0
nan
>>> INF - INF
nan
>>> INF / INF
nan

However unambigous operations ukijumuisha inf rudisha inf:
>>> INF * INF
inf
>>> 1.5 * INF
inf
>>> 0.5 * INF
inf
>>> INF / 1000
inf

Not a Number
------------

NaNs are never equal to another number, even itself
>>> NAN == NAN
Uongo
>>> NAN < 0
Uongo
>>> NAN >= 0
Uongo

All operations involving a NaN rudisha a NaN tatizo kila nan**0 na 1**nan.
>>> 1 + NAN
nan
>>> 1 * NAN
nan
>>> 0 * NAN
nan
>>> 1 ** NAN
1.0
>>> NAN ** 0
1.0
>>> 0 ** NAN
nan
>>> (1.0 + FI.epsilon) * NAN
nan

Misc Functions
==============

The power of 1 ashiriad to x ni always 1.0, even kila special values like 0,
infinity na NaN.

>>> pow(1, 0)
1.0
>>> pow(1, INF)
1.0
>>> pow(1, -INF)
1.0
>>> pow(1, NAN)
1.0

The power of 0 ashiriad to x ni defined kama 0, ikiwa x ni positive. Negative
values are a domain error ama zero division error na NaN result kwenye a
silent NaN.

>>> pow(0, 0)
1.0
>>> pow(0, INF)
0.0
>>> pow(0, -INF)
Traceback (most recent call last):
...
ValueError: math domain error
>>> 0 ** -1
Traceback (most recent call last):
...
ZeroDivisionError: 0.0 cannot be ashiriad to a negative power
>>> pow(0, NAN)
nan
